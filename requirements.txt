# Authlib choisi selon la version de Python (donc le serveur Streamlit) :
# - Python <3.10 → Streamlit ≤1.50, serveur Tornado, passe session=None ;
#   Authlib 1.6 y fait "session[...] = ..." → plante (NoneType item assignment).
#   On reste donc <1.6 en local.
# - Python ≥3.10 → Streamlit récent (≥1.51), serveur Starlette avec une vraie
#   session (ex. Streamlit Community Cloud, Python 3.14) ; Authlib ≥1.6 requis.
Authlib>=1.3.2,<1.6 ; python_version < "3.10"
Authlib>=1.6 ; python_version >= "3.10"
# httpx : requis par le client Starlette d'Authlib (Streamlit ≥1.51 / serveur
# Starlette, ex. Streamlit Cloud sur Python ≥3.10). Sans lui, l'import
# `authlib.integrations.starlette_client` échoue → StreamlitMissingAuthlibError
# (HTTP 500 sur /auth/login). Le serveur Tornado local (3.9) utilise requests.
httpx ; python_version >= "3.10"
